# 后端开发手册

## 1. 启动与装配

后端入口固定为 `app/backend/app.py`。

启动流程：

1. 加载 `.env`
2. 初始化 Flask
3. 初始化数据库
4. 同步身份配置
5. 启动 worker
6. 注册所有路由

## 2. 目录约定

```text
app/backend/
├── app.py
├── modules/
│   ├── auth/
│   ├── ai/
│   ├── feature3/
│   ├── pages/
│   ├── diagnostics/
│   └── shared/
└── tests/
```

约定：业务实现放在 `modules/`，不要在 backend 根目录新增兼容中转文件。

## 3. 数据与配置

- 数据库：`feature3.db`
- 知识库状态：`kb_meta.json`, `kb_state.json`, `kb_texts.json`
- 环境变量：`.env`

## 4. API 开发建议

- 业务逻辑放 `service.py`
- 参数与响应包装放 `routes.py`
- 数据访问统一走 `modules/shared/db.py`

## 5. 调试建议

优先检查：

1. 当前目录是否在 `app/backend`
2. Python 环境是否正确
3. `.env` 是否存在
4. 5000 端口是否被占用
